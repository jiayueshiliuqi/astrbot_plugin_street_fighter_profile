# astrbot_plugin_street_fighter_profile

街霸 6 玩家信息查询插件。

支持查询玩家段位、胜率、角色战绩以及搜索 CFN 玩家。

## 功能

- 通过 Short ID 查询玩家信息
- 查看最高段位、LP、MR 和 MR 排名
- 查看总胜率和排位胜率
- 查询指定角色的段位与战绩
- 查询全部角色数据
- 通过 CFN 名称搜索玩家
- 支持搜索结果翻页
- 支持全局人格管理
- 支持清空当前会话记忆

## 安装

将插件文件夹放入 AstrBot 的插件目录，然后在管理面板中重载插件或重启 AstrBot。

插件目录至少需要包含：

```text
astrbot_plugin_street_fighter_profile/
├── main.py
├── sf6_client.py
├── metadata.yaml
├── cookie.txt
└── README.md
```

## 安装依赖

插件需要安装：

```bash
pip install requests
```

请在 AstrBot 实际使用的 Python 环境中安装依赖。

## 配置 Cookie

> 重要：使用查询功能前，必须在 `main.py` 所在目录设置 `cookie.txt`。

### 1. 登录官网

使用浏览器打开街霸 6 官网：

```text
https://www.streetfighter.com/6/buckler/zh-hans
```

登录自己的 Capcom ID，并确认可以正常进入页面。

### 2. 打开开发者工具

登录后按下：

```text
F12
```

进入：

```text
Network
```

中文浏览器中一般显示为“网络”。

如果没有显示内容，请保持开发者工具打开并刷新网页。

### 3. 查找相关记录

在 Network 的筛选框中输入：

```text
sf
```

也可以输入：

```text
buckler
```

然后选择一个来自以下网站的相关记录：

```text
www.streetfighter.com
```

### 4. 复制 Cookie

点击找到的记录，在右侧依次找到：

```text
Headers
Request Headers
Cookie
```

复制 `Cookie` 后面的完整内容。

内容类似：

```text
CookieConsent={xxxx
```

注意：

- 只复制 Cookie 后面的值
- 不要复制 `Cookie:` 字样
- 不要复制整个 Headers
- Cookie 通常较长，请确保没有遗漏
- 不要将 Cookie 发送给其他人

### 5. 创建 cookie.txt

在 `main.py` 所在目录：

```text
cookie.txt
```

将刚才复制的 Cookie 完整粘贴到文件中，保持一行即可。

配置后的目录结构：

```text
astrbot_plugin_street_fighter_profile/
├── main.py
├── sf6_client.py
├── cookie.txt
├── metadata.yaml
└── README.md
```

完成后重载插件或重启 AstrBot。

## Cookie 安全提醒

`cookie.txt` 包含你的登录信息，请妥善保管。

请勿：

- 将 `cookie.txt` 上传到 GitHub
- 将 Cookie 发送到群聊或其他公开位置
- 在截图、Issue 或日志中展示 Cookie
- 将真实 Cookie 写入代码或 README
- 与其他人共享自己的 Cookie

建议在 `.gitignore` 中添加：

```gitignore
cookie.txt
personas.json
debug/
__pycache__/
*.pyc
```

如果 Cookie 不慎泄露，请立即退出官网登录状态，并重新登录以更新登录信息。

## 使用方法

### 查询玩家

通过 Short ID 查询：

```text
/查询 <short_id>
```

示例：

```text
/查询 1234567890
```

查询结果包括玩家段位、角色、LP 或 MR，以及胜率等信息。

### 查询指定角色

```text
/查询 <short_id> 角色 <角色名>
```

示例：

```text
/查询 1234567890 角色 杰米
```

也可以使用英文角色名：

```text
/查询 1234567890 角色 jamie
```

### 查询全部角色

```text
/查询 <short_id> 角色 全部
```

示例：

```text
/查询 1234567890 角色 全部
```

### 搜索 CFN 玩家

```text
/查询 <CFN名称>
```

示例：

```text
/查询 TestPlayer
```

搜索名称至少需要 4 个字符。

搜索完成后，可以使用序号查看玩家详情：

```text
/查询 1
```

可选择的序号以当前显示的搜索结果为准。

### 搜索结果翻页

查看下一页：

```text
/下一页
```

返回上一页：

```text
/上一页
```

插件重载或 AstrBot 重启后，未完成的搜索状态会被清除。

### 查看帮助

```text
/help
```

## 人格管理

插件支持设置一个全局人格。启用后，所有群聊和私聊的新消息都会使用该人格。

### 添加人格

管理员命令：

```text
/人格添加 <名称> <提示词>
```

示例：

```text
/人格添加 大小姐 你是一位金发傲娇大小姐，说话简短自然，不使用Markdown、表情或动作描写。
```

限制：

- 人格名称不能超过 32 个字符
- 人格提示词不能超过 8000 个字符
- 添加同名人格会覆盖原有内容

### 删除人格

管理员命令：

```text
/人格删除 <名称>
```

示例：

```text
/人格删除 大小姐
```

删除当前正在使用的人格后，将恢复 AstrBot 默认人格。

### 查看人格列表

```text
/人格查询
```

查看指定人格内容：

```text
/人格查询 <名称>
```

示例：

```text
/人格查询 大小姐
```

### 设置全局人格

管理员命令：

```text
/人格设置 <名称>
```

示例：

```text
/人格设置 大小姐
```

设置成功后，所有群聊和私聊的后续消息都会使用该人格。

### 关闭全局人格

管理员命令：

```text
/人格设置 关闭
```

也可以使用：

```text
/人格设置 默认
```

或者：

```text
/人格设置 取消
```

关闭后将恢复 AstrBot 默认人格。

### 查看当前人格

```text
/人格当前
```

### 查看人格帮助

```text
/help_p
```

人格数据保存在插件目录的：

```text
personas.json
```

该文件由插件自动创建。

## 清空会话记忆

管理员命令：

```text
/记忆清空
```

该命令只清空执行命令所在会话的聊天记录。

清空后：

- 当前会话以前的聊天内容将被清除
- 当前全局人格保持不变
- 其他群聊或私聊不会受到影响
- 下一条消息会使用新的聊天上下文

## 常见问题

### 提示查询失败

请依次检查：

1. `cookie.txt` 是否与 `main.py` 位于同一目录
2. `cookie.txt` 是否为空
3. Cookie 是否复制完整
4. 官网是否仍然处于登录状态
5. AstrBot 所在设备是否可以正常访问官网
6. 更新 `cookie.txt` 后是否重载了插件

### Cookie 失效怎么办

重新登录官网，按前面的步骤获取最新 Cookie，然后覆盖：

```text
cookie.txt
```

覆盖完成后，需要重载插件或重启 AstrBot。

### 更新 Cookie 后仍然查询失败

插件启动后会保留已经读取的 Cookie。仅修改文件可能不会立即生效，请重载插件或重启 AstrBot。

### 搜索不到玩家

请确认：

- 搜索名称至少包含 4 个字符
- CFN 名称拼写正确
- 输入的是 CFN 名称，而不是其他平台昵称
- 官网当前能够正常显示该玩家

### 无法识别角色名称

可以尝试使用中文名或英文名。

例如：

```text
/查询 1234567890 角色 隆
```

```text
/查询 1234567890 角色 ryu
```

查询所有角色：

```text
/查询 1234567890 角色 全部
```

### 人格设置后回复风格没有明显变化

新人格会从后续消息开始生效，但当前会话以前的聊天内容仍可能影响回复。

管理员可以执行：

```text
/记忆清空
```

然后重新发送消息。

## 数据说明

- 查询结果以当前可获取的信息为准
- 部分战绩可能只显示近期数据
- 不同页面的数据更新时间可能存在差异
- 未定级角色可能不会显示完整段位信息
- 显示结果仅供参考，以游戏内实际信息为准
- 游戏或网站更新后，部分功能可能需要同步适配
